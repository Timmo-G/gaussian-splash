## 0. 3DGS训练的本质是什么？

3DGS训练的目标：给定多张照片，找到最优的数百万个高斯基元参数，使得渲染结果与真实照片尽可能接近。

这是一个**优化问题**：
```
min_{μ, q, s, α, sh} L(渲染图像, 真实图像)
```

其中：
- μ：每个高斯的位置（3D向量）
- q：旋转（四元数）
- s：缩放（3D向量）
- α：不透明度（标量）
- sh：球谐函数系数（颜色）

**优化方法**：梯度下降（具体用Adam优化器）

**关键问题**：如何计算损失函数对每个参数的梯度？→ 反向传播

## 2.1 从一元函数到多元函数

### 一元函数的导数（回顾）

导数 f'(x) 的几何意义：函数在 x 处的切线斜率。

```
f'(x) = lim_{Δx→0} [f(x+Δx) - f(x)] / Δx
```

**常用求导规则**：
- (xⁿ)' = nxⁿ⁻¹
- (eˣ)' = eˣ
- (ln x)' = 1/x
- (sin x)' = cos x
- (f·g)' = f'g + fg'（乘积法则）
- (f(g(x)))' = f'(g(x))·g'(x)（链式法则）

### 偏导数：固定其他变量

对于 f(x, y)，偏导数 ∂f/∂x 的意思是：**固定 y 不变**，只让 x 变化时，f 的变化率。

```
∂f/∂x = lim_{Δx→0} [f(x+Δx, y) - f(x, y)] / Δx
```

**计算方法**：把其他变量当常数，对目标变量用普通求导规则。

**例子**：f(x, y) = x²y + sin(xy)
- ∂f/∂x = 2xy + y·cos(xy)（把 y 当常数）
- ∂f/∂y = x² + x·cos(xy)（把 x 当常数）

### 梯度：所有偏导数的集合

梯度 ∇f 是一个**向量**，包含所有偏导数：

```
∇f(x, y, z) = (∂f/∂x, ∂f/∂y, ∂f/∂z)
```

**几何意义**：梯度向量指向函数值**增长最快**的方向，梯度的模是该方向的增长速率。

**关键性质**：
- 梯度方向 = 函数值增长最快的方向
- **负梯度方向** = 函数值下降最快的方向（梯度下降的依据！）
- 在极值点处，梯度 = 0（必要条件）
- 梯度垂直于等值面（等高线）

## 2.2 链式法则 —— 反向传播的数学基础

### 一元链式法则

如果 y = f(g(x))，则：
```
dy/dx = f'(g(x)) · g'(x) = (df/dg) · (dg/dx)
```

**直觉**：复合函数的导数 = 外层导数 × 内层导数。

**例子**：y = sin(x²)
- 令 u = x²，则 y = sin(u)
- dy/du = cos(u) = cos(x²)
- du/dx = 2x
- dy/dx = cos(x²) · 2x

### 多元链式法则

如果 z = f(x, y)，而 x = g(t)，y = h(t)，则：
```
dz/dt = (∂z/∂x)(dx/dt) + (∂z/∂y)(dy/dt)
```

**直觉**：t 的变化通过两条路径影响 z（通过 x 和通过 y），总影响是两条路径的叠加。

### 计算图：反向传播的图形化理解

计算图把复杂的函数分解成简单操作的有向图：
- **节点**：变量或操作
- **边**：数据流向

**前向传播**：从输入到输出，计算函数值。
**反向传播**：从输出到输入，用链式法则计算梯度。

**例子**：f(x, y) = (x + y) · sin(x)

```
计算图：
x ──┬──→ [+] ──→ z₁ = x+y ──→ [×] ──→ output
y ──┘                              ↑
x ──→ [sin] ──→ z₂ = sin(x) ──────┘
```

反向传播（设 ∂L/∂output = 1）：
```
∂L/∂z₁ = ∂L/∂output · ∂output/∂z₁ = 1 · sin(x) = sin(x)
∂L/∂z₂ = ∂L/∂output · ∂output/∂z₂ = 1 · z₁ = x+y
∂L/∂x  = ∂L/∂z₁ · ∂z₁/∂x + ∂L/∂z₂ · ∂z₂/∂x
        = sin(x) · 1 + (x+y) · cos(x)
∂L/∂y  = ∂L/∂z₁ · ∂z₁/∂y = sin(x) · 1 = sin(x)
```

### 3DGS的梯度流

3DGS的计算图（简化版）：

```
高斯参数 (μ, q, s, α, sh)
    ↓ 参数化
协方差矩阵 Σ = R·S·Sᵀ·Rᵀ
    ↓ 投影
2D协方差 Σ' = J·Σ·Jᵀ，2D均值 μ'
    ↓ 高斯值
像素贡献 αᵢ = opacity · G(pixel; μ', Σ')
    ↓ Alpha合成
渲染颜色 C = Σᵢ Tᵢ · αᵢ · cᵢ
    ↓ 损失
L = (1-λ)·L₁ + λ·L_SSIM
```

梯度从 L 反向流回每个参数，每一步都用链式法则。

**与官方代码的对应**：
```python
# gaussian_renderer/__init__.py
# render() 函数执行前向传播
# 梯度由 diff-gaussian-rasterization CUDA扩展自动计算

# train.py 中的训练循环：
loss = (1.0 - opt.lambda_dssim) * Ll1 + opt.lambda_dssim * (1.0 - ssim(image, gt_image))
loss.backward()  # 反向传播，计算所有参数的梯度
optimizer.step() # Adam更新参数
```

## 2.3 梯度下降 —— 完整推导

### 基本思想

从当前参数出发，沿着**负梯度方向**走一小步：

```
θ ← θ - α · ∇_θ L(θ)
```

**为什么沿负梯度方向？**

一阶泰勒展开：f(θ + Δθ) ≈ f(θ) + ∇f(θ)ᵀ · Δθ

要使 f 减小，需要 ∇f(θ)ᵀ · Δθ < 0。

在 |Δθ| = ε（步长固定）的约束下，使 ∇f(θ)ᵀ · Δθ 最小的方向是 Δθ = -ε · ∇f(θ)/|∇f(θ)|。

所以，**负梯度方向是函数值下降最快的方向**。

### 学习率的影响

| 学习率 α | 效果 | 原因 |
|---------|------|------|
| 太大 | 震荡，可能发散 | 步子太大，跨过极小值 |
| 太小 | 收敛极慢 | 步子太小，需要很多步 |
| 合适 | 稳定收敛 | 平衡速度和稳定性 |

**如何选择学习率**：
- 从较小值开始（如 1e-3）
- 观察损失曲线：下降太慢→增大，震荡→减小
- 使用学习率调度（如余弦退火）

### 病态问题：为什么需要自适应学习率

考虑 f(x, y) = 0.5(x² + 100y²)：
- x 方向梯度：∂f/∂x = x（量级~1）
- y 方向梯度：∂f/∂y = 100y（量级~100）

如果用同一个学习率：
- 对 y 方向合适的学习率（~0.01）对 x 方向太小（收敛慢）
- 对 x 方向合适的学习率（~1）对 y 方向太大（震荡）

**解决方案**：为每个参数自适应调整学习率 → Adam

## 2.4 Adam优化器 —— 完整推导

### 动机

Adam（Adaptive Moment Estimation）结合了两个思想：
1. **动量**（Momentum）：记住过去的梯度方向，避免震荡
2. **自适应学习率**（RMSProp）：梯度大的参数用小学习率，梯度小的参数用大学习率

### 完整算法

```
初始化：m₀ = 0，v₀ = 0，t = 0
超参数：α=0.001，β₁=0.9，β₂=0.999，ε=1e-8

每步更新：
  t ← t + 1
  gₜ = ∇_θ L(θₜ₋₁)          # 计算梯度

  mₜ = β₁·mₜ₋₁ + (1-β₁)·gₜ  # 一阶矩（梯度的指数移动平均）
  vₜ = β₂·vₜ₋₁ + (1-β₂)·gₜ² # 二阶矩（梯度平方的指数移动平均）

  m̂ₜ = mₜ / (1 - β₁ᵗ)        # 偏差修正
  v̂ₜ = vₜ / (1 - β₂ᵗ)        # 偏差修正

  θₜ = θₜ₋₁ - α · m̂ₜ / (√v̂ₜ + ε)  # 参数更新
```

### 各组件的直觉

**一阶矩 mₜ（动量）**：
- β₁=0.9 意味着：当前梯度占10%，历史梯度占90%
- 效果：平滑梯度方向，减少震荡
- 类比：小球在碗里滚动，有惯性

**二阶矩 vₜ（自适应学习率）**：
- β₂=0.999 意味着：当前梯度平方占0.1%，历史占99.9%
- 效果：估计每个参数的梯度"幅度"
- 有效学习率 = α / √v̂ ≈ α / (梯度的RMS)

**偏差修正**：
- 初始时 m₀=v₀=0，导致早期估计偏低
- 修正：m̂ = m / (1 - β₁ᵗ)，t=1时修正因子 = 1/(1-0.9) = 10

**有效学习率**：
```
有效lr = α / √v̂ ≈ α / (梯度的均方根)
```
- 梯度大 → v̂ 大 → 有效lr 小（防止震荡）
- 梯度小 → v̂ 小 → 有效lr 大（加速收敛）

### 3DGS中Adam的配置

```python
# train.py 中的优化器设置
l = [
    {'params': [self._xyz],           'lr': 0.00016, "name": "xyz"},
    {'params': [self._features_dc],   'lr': 0.0025,  "name": "f_dc"},
    {'params': [self._features_rest], 'lr': 0.0025/20, "name": "f_rest"},
    {'params': [self._opacity],       'lr': 0.05,    "name": "opacity"},
    {'params': [self._scaling],       'lr': 0.005,   "name": "scaling"},
    {'params': [self._rotation],      'lr': 0.001,   "name": "rotation"},
]
self.optimizer = torch.optim.Adam(l, lr=0.0, eps=1e-15)
```

注意：不同参数使用**不同的学习率**！位置(xyz)学习率最小，不透明度(opacity)学习率最大。

## 2.5 损失函数 —— 深入理解

### L1损失（MAE）

```
L₁ = (1/N) Σᵢ |yᵢ - ŷᵢ|
```

**梯度**：∂L₁/∂ŷᵢ = -sign(yᵢ - ŷᵢ) / N

**特点**：
- 梯度恒定（±1/N），不受误差大小影响
- 对异常值不敏感（大误差不会产生超大梯度）
- 在误差=0处不可微（但可以用次梯度）

### L2损失（MSE）

```
L₂ = (1/N) Σᵢ (yᵢ - ŷᵢ)²
```

**梯度**：∂L₂/∂ŷᵢ = -2(yᵢ - ŷᵢ) / N

**特点**：
- 梯度与误差成正比（大误差→大梯度→快速修正）
- 对异常值敏感（大误差被平方放大）
- 处处可微

### SSIM（结构相似性）—— 完整推导

SSIM从三个维度衡量图像相似性：

```
SSIM(x, y) = l(x,y) · c(x,y) · s(x,y)
```

**亮度相似性**：
```
l(x,y) = (2μₓμᵧ + C₁) / (μₓ² + μᵧ² + C₁)
```
其中 μₓ, μᵧ 是局部均值，C₁=(0.01·L)²（L是像素值范围）

**对比度相似性**：
```
c(x,y) = (2σₓσᵧ + C₂) / (σₓ² + σᵧ² + C₂)
```
其中 σₓ, σᵧ 是局部标准差

**结构相似性**：
```
s(x,y) = (σₓᵧ + C₃) / (σₓσᵧ + C₃)
```
其中 σₓᵧ 是局部协方差，C₃=C₂/2

**为什么SSIM比L2更好？**
- L2：像素级比较，不考虑空间结构
- SSIM：考虑局部统计特性（均值、方差、协方差），更符合人眼感知
- 例如：整体亮度偏移对L2影响大，对SSIM影响小（亮度项接近1）

### 3DGS的训练损失

```
L = (1 - λ) · L₁ + λ · L_D-SSIM
```

其中 L_D-SSIM = (1 - SSIM) / 2，λ = 0.2（默认值）

**为什么这样组合？**
- L₁：训练稳定，梯度恒定，不会因为异常值产生爆炸梯度
- SSIM：感知质量好，但梯度计算复杂
- 组合：兼顾稳定性和感知质量

**与官方代码的对应**：
```python
# train.py
from utils.loss_utils import l1_loss, ssim

Ll1 = l1_loss(image, gt_image)
loss = (1.0 - opt.lambda_dssim) * Ll1 + \
       opt.lambda_dssim * (1.0 - ssim(image, gt_image))
```

## 2.6 3DGS训练的完整梯度流

理解梯度如何从损失函数流回到每个高斯参数，是理解3DGS训练的关键。

### 简化的梯度流分析

**从损失到渲染颜色**：
```
∂L/∂C = ∂L/∂L₁ · ∂L₁/∂C + ∂L/∂SSIM · ∂SSIM/∂C
```

**从渲染颜色到每个高斯的贡献**（Alpha合成的梯度）：
```
C = Σᵢ Tᵢ · αᵢ · cᵢ

∂C/∂cᵢ = Tᵢ · αᵢ                    # 颜色梯度
∂C/∂αᵢ = Tᵢ · cᵢ - Σⱼ>ᵢ Tⱼ·αⱼ·cⱼ/(1-αᵢ)  # 不透明度梯度（复杂！）
```

**从不透明度到高斯参数**：
```
αᵢ = opacity · G(pixel; μ', Σ')

∂αᵢ/∂μ' = αᵢ · Σ'⁻¹(pixel - μ')    # 位置梯度
∂αᵢ/∂Σ' = αᵢ · (-1/2) · Σ'⁻¹(pixel-μ')(pixel-μ')ᵀΣ'⁻¹  # 协方差梯度
```

**从2D协方差到3D参数**（链式法则）：
```
∂L/∂Σ = ∂L/∂Σ' · ∂Σ'/∂Σ = ∂L/∂Σ' · JᵀJ（近似）
∂L/∂q = ∂L/∂Σ · ∂Σ/∂R · ∂R/∂q
∂L/∂s = ∂L/∂Σ · ∂Σ/∂S · ∂S/∂s
```

这些梯度在官方代码中由CUDA扩展（`diff-gaussian-rasterization`）高效计算。

## 常见误区

**误区1**：梯度下降一定能找到全局最优
- 错误！梯度下降只能找到局部极小值
- 3DGS的损失函数有很多局部极小值，但实践中效果很好

**误区2**：学习率越小越好
- 错误！学习率太小会导致收敛极慢，甚至陷入局部极小值
- 需要在速度和稳定性之间平衡

**误区3**：Adam总是比SGD好
- 不一定！Adam在某些问题上泛化性不如SGD
- 3DGS选择Adam是因为参数数量巨大（数百万），需要自适应学习率

**误区4**：反向传播需要手动计算
- 现代深度学习框架（PyTorch）自动计算梯度（自动微分）
- 但理解反向传播原理对调试和改进模型至关重要
