## 4.1 相机模型精讲

### 针孔相机模型

最简单的相机模型：光线通过一个无限小的孔（针孔）投影到像平面。

**坐标系变换链**：
```
世界坐标 P_w → [R|t] → 相机坐标 P_c → K → 图像坐标 p
```

### 内参矩阵 K —— 完整推导

```
K = [[fx,  0, cx],
     [ 0, fy, cy],
     [ 0,  0,  1]]
```

**fx, fy 的物理意义**：

设相机的物理焦距为 f（毫米），像素宽度为 dx（毫米/像素），则：
```
fx = f / dx（像素单位的焦距）
```

**为什么 fx ≠ fy？**

如果像素不是正方形（非方形像素），则 dx ≠ dy，导致 fx ≠ fy。现代相机通常 fx ≈ fy。

**cx, cy 的物理意义**：

主点（光轴与像平面的交点）在图像中的位置。理想情况下 cx=W/2, cy=H/2，但实际相机有偏差。

**投影公式推导**：

设3D点在相机坐标系中为 P_c = (X, Y, Z)，则：
1. 透视除法：X' = X/Z，Y' = Y/Z（归一化坐标）
2. 乘以焦距：u' = fx·X'，v' = fy·Y'
3. 加主点偏移：u = u' + cx，v = v' + cy

矩阵形式：
```
[u]   [fx  0  cx] [X/Z]
[v] = [ 0 fy  cy] [Y/Z]
[1]   [ 0  0   1] [ 1 ]
```

等价于：
```
Z·[u, v, 1]ᵀ = K · [X, Y, Z]ᵀ
```

### 外参矩阵 [R|t] —— 注意事项

```
P_c = R · P_w + t
```

**常见混淆**：t 不是相机在世界坐标系中的位置！

- t：世界原点在相机坐标系中的位置
- 相机在世界坐标系中的位置：C = -Rᵀt

**验证**：当相机在世界原点（C=0）时，t = -R·0 = 0 ✓

**3DGS中的外参**：
```python
# scene/cameras.py
# self.R: 旋转矩阵（世界→相机）
# self.T: 平移向量 t
# 相机位置：-R^T @ T
```

### 畸变模型（简介）

真实镜头存在畸变，主要有：
- **径向畸变**：图像边缘的桶形或枕形变形
  - k₁, k₂, k₃：径向畸变系数
- **切向畸变**：镜头与像平面不完全平行
  - p₁, p₂：切向畸变系数

**畸变校正公式**：
```
x_distorted = x(1 + k₁r² + k₂r⁴) + 2p₁xy + p₂(r² + 2x²)
```

3DGS训练时需要已知相机内外参（通常由COLMAP估计），COLMAP会同时估计畸变参数。

## 4.2 光线投射与体积渲染

### 光线方程

从相机出发，穿过像素 (u, v) 的光线：
```
r(t) = o + t·d
```

- **o**：光线起点（相机位置）
- **d**：光线方向（归一化）
- **t**：沿光线的参数（深度）

**光线方向的计算**：
```
d = normalize(K⁻¹ · [u, v, 1]ᵀ)
```

### NeRF的体积渲染方程 —— 完整推导

NeRF用体积渲染来合成图像：
```
C(r) = ∫[t_n to t_f] T(t) · σ(r(t)) · c(r(t), d) dt
```

其中透射率：
```
T(t) = exp(-∫[t_n to t] σ(r(s)) ds)
```

**物理意义**：
- σ(x)：体积密度（光子被吸收的概率密度）
- c(x, d)：颜色（在位置x、方向d处的辐射亮度）
- T(t)：从 t_n 到 t 的透射率（光线能穿透的概率）

**离散近似**（实际计算）：

将光线分成 N 段，每段长度 δᵢ：
```
αᵢ = 1 - exp(-σᵢδᵢ)  （第i段的不透明度）
Tᵢ = Πⱼ<ᵢ (1 - αⱼ)   （到第i段的透射率）
C ≈ Σᵢ Tᵢ · αᵢ · cᵢ
```

### 3DGS与NeRF的区别

| 方面 | NeRF | 3DGS |
|------|------|------|
| 表示 | 隐式（神经网络） | 显式（高斯基元） |
| 渲染 | 沿光线积分 | 投影+alpha合成 |
| 速度 | 慢（分钟级） | 快（实时） |
| 编辑 | 困难 | 相对容易 |
| 内存 | 小（网络参数） | 大（数百万高斯） |

**3DGS快的原因**：
1. 不需要沿光线采样（每像素64-128次网络查询）
2. 高斯投影是解析的（不需要迭代）
3. Tile-based渲染适合GPU并行化
4. 提前终止（透射率接近0时停止）

## 4.3 Alpha合成（Alpha Compositing）

### 前向Alpha合成公式

3DGS渲染每个像素颜色的核心公式：
```
C = Σᵢ cᵢ · αᵢ · Πⱼ<ᵢ (1 - αⱼ)
```

其中：
- **cᵢ**：第 i 个高斯的颜色（由球谐函数计算）
- **αᵢ**：第 i 个高斯在该像素的不透明度
  ```
  αᵢ = opacity_i · G(pixel; μ'ᵢ, Σ'ᵢ)
  ```
  其中 G 是2D高斯值（最大值为1）
- **Πⱼ<ᵢ (1 - αⱼ)**：前面所有高斯的透射率

### 与NeRF体积渲染的对应关系

| NeRF | 3DGS |
|------|------|
| αᵢ = 1 - exp(-σᵢδᵢ) | αᵢ = opacity · G(pixel) |
| Tᵢ = Πⱼ<ᵢ (1-αⱼ) | Tᵢ = Πⱼ<ᵢ (1-αⱼ) |
| 沿光线积分 | 按深度排序后叠加 |

**关键区别**：NeRF的 αᵢ 来自神经网络预测的密度，3DGS的 αᵢ 来自2D高斯值。

### 可微性分析

Alpha合成对所有参数可微：

```
∂C/∂cᵢ = Tᵢ · αᵢ                    # 颜色梯度
∂C/∂αᵢ = Tᵢ · cᵢ - Σⱼ>ᵢ Tⱼ·αⱼ·cⱼ/(1-αᵢ)  # 不透明度梯度
```

这些梯度在官方代码中由CUDA扩展高效计算。

## 4.4 SfM与点云（COLMAP）

### Structure from Motion（SfM）原理

从多张照片恢复相机位姿和稀疏点云：

1. **特征提取**：在每张图片中找特征点（SIFT等）
2. **特征匹配**：找到不同图片中的对应点
3. **增量式重建**：逐步添加图片，估计相机位姿和3D点位置

**数学基础**：对极几何（Epipolar Geometry）

两张图片中的对应点满足：
```
p₂ᵀ · E · p₁ = 0
```

其中 E 是本质矩阵（Essential Matrix），包含相机间的旋转和平移信息。

### COLMAP工作流程

```
输入：多张建筑照片
  ↓ 特征提取（SIFT）
  ↓ 特征匹配（暴力匹配或词袋模型）
  ↓ 增量式SfM重建
输出：
  - cameras.txt：相机内参
  - images.txt：每张图片的外参（R, t）
  - points3D.txt：稀疏点云（3D点坐标 + 颜色）
```

### 3DGS中的使用

3DGS训练的输入就是COLMAP的输出：
- **相机参数**：用于投影变换（K, R, t）
- **稀疏点云**：用于初始化高斯基元的位置

**初始化策略**：
```python
# scene/gaussian_model.py create_from_pcd()
# 对每个COLMAP点云点，创建一个高斯基元
# 初始位置 = 点云坐标
# 初始颜色 = 点云颜色（转换为球谐系数）
# 初始缩放 = 基于邻近点距离估计
# 初始旋转 = 单位四元数（无旋转）
# 初始不透明度 = 0.1（较小，训练中逐渐增大）
```

## 4.5 Tile-based渲染 —— 3DGS的加速策略

### 为什么需要Tile-based渲染？

朴素的渲染方法：对每个像素，遍历所有高斯，计算贡献。

**问题**：N个像素 × M个高斯 = O(NM) 操作，太慢。

**Tile-based渲染**：将图像分成 16×16 的小块（tile），每个tile独立处理。

### 算法流程

```
步骤1：预处理（对每个高斯）
  - 投影到2D（计算μ', Σ'）
  - 计算影响的tile范围（3σ范围内的tile）
  - 为每个(tile, 高斯)对创建排序键：key = tile_id * 2^32 + depth

步骤2：排序（GPU并行基数排序）
  - 按排序键排序所有(tile, 高斯)对
  - 结果：同一tile内的高斯按深度排序

步骤3：渲染（每个tile独立，GPU并行）
  - 每个tile的线程块处理该tile内的所有像素
  - 按深度顺序执行alpha合成
  - 提前终止：当透射率T < 0.0001时停止
```

### 与官方代码的对应

```python
# submodules/diff-gaussian-rasterization/cuda_rasterizer/forward.cu

# preprocessCUDA(): 投影 + 计算tile范围
# 对应Python接口：rasterize_gaussians()

# renderCUDA(): 每个tile的alpha合成
# 关键优化：
#   1. 共享内存（shared memory）缓存tile内的高斯数据
#   2. 提前终止（early termination）
#   3. 反向传播时重用前向传播的中间结果
```

## 常见误区

**误区1**：t 是相机在世界坐标系中的位置
- 错误！t 是世界原点在相机坐标系中的位置
- 相机位置 C = -Rᵀt

**误区2**：3DGS的深度排序是每像素的
- 错误！3DGS对所有高斯做全局深度排序（按相机坐标系中的z值）
- 这是一个近似，对于透明物体可能有排序错误

**误区3**：Tile-based渲染只是为了速度
- 不完全对！Tile-based渲染也使得反向传播更高效（每个tile独立，梯度不跨tile）
